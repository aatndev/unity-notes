ROTATION NOTES
---------------
1. Standard Rotation and Vector3: [rotate from where the object is rn]

transform.Rotate(xAxis, yAxis, zAxis); //turn from where it is
eg: transform.Rotate(0, 10, 0); // turn the object 10 degree on y axis
//this is similar to Space.Self - by default its Space.Self

transform.Rotate(Vector3.up * 10, Space.World); //Turn 10° around the world's up direction.
-- Vector3.up : (0, 1, 0) times 10 is: (0, 10, 0) - rotates in world direction

the above command does same thing that below command do:
playerBodyPart.transform.Rotate(new Vector3(0, 10, 0), Space.World);

2. Euler Angles: Rotation using Degrees .. [be in that position. or rotate to that angle]
transform.eulerAngles = new Vector3(30,90,0); -- this is not rotate that much. this is rotate to that angle

3. Quaternion
''transform.rotation = Quaternion.Euler(30,90,0);'' does same thing that ''transform.eulerAngles = new Vector3(30,90,0);'' do

''Quaternion.identity is (0, 0, 0, 1)'':
x, y, z = 0, and w = 1.. w is cos (angle / 2) = cos 90 = 0.. so rotation = 0
similar to : ''transform.eulerAngles = new Vector3(0,0,0);''

transform.localRotation = Quaternion.Euler(xAxis, yAxis, zAxis);

4. Local Rotation and Quaternion
transform.localRotation = Quaternion.Euler(0, 30, 0); IS SAME AS: transform.localRotation = Quaternion.Euler(Vector3.up * 30);
//where 30 is the angle. 
This is : Set your local rotation to this exact orientation (0, 30 degree, 0) (relative to your parent). -- means inside ur parent.

public class Drag : MonoBehaviour, IBeginDragHandler, IDragHandler, IEndDragHandler
{
    public RectTransform limitCircle; // The circle UI you want to stay inside
    private RectTransform spoonRect;

    void Start()
    {
        spoonRect = GetComponent<RectTransform>();
    }

    public void OnBeginDrag(PointerEventData eventData)
    {
        // Optional: play sound or animation
    }

    public void OnDrag(PointerEventData eventData)
    {
        // Move spoon to mouse
        spoonRect.position = eventData.position;

        // Clamp spoon inside the circle
        KeepInsideCircle();
    }

    public void OnEndDrag(PointerEventData eventData)
    {
        // Optional
    }

    void KeepInsideCircle()
    {
        Vector2 center = limitCircle.position;
        float radius = limitCircle.rect.width * 0.5f;

        Vector2 direction = spoonRect.position - center;

        if (direction.magnitude > radius)
        {
            spoonRect.position = center + direction.normalized * radius;
        }
    }
}

DOT PRODUCT AND APPLICATOIN
---------------------------
---------------------------
Vector3.Dot(a, b) means
a.x * b.x + a.y * b.y + a.z * b.z


COMPARISON REAL LIFE APPL:-

Vector3 toTarget = realPath.transform.position - plane.transform.position;

if (Vector3.Dot(plane.transform.forward, toTarget) < 0)
{
    // The realPath is behind the plane (plane has passed it)
}
else
{
    // The realPath is in front of the plane
}

EXPL:

- Vector3 toTarget = realPath.transform.position - plane.transform.position;
> has plane------>realPath [distance and direction]

-Vector3.Dot(plane.transform.forward, toTarget)
plane.transform.forward > has plane's forward direction

Therefore if,
> Vector3.Dot(plane.transform.forward, toTarget) > 0 
Then: both are facing same direction - meaning realPath is in front of plane

> Vector3.Dot(plane.transform.forward, toTarget) == 0
Then: both are at the same place because vectors are perpendicular

> Vector3.Dot(plane.transform.forward, toTarget) < 0
Then: realPath is behind the plane because realPath's direction is towards the plane and toTargets direction is towards the realPath
